# LEV / LIQ

一个给个人使用的韩股、日股、美股杠杆与流动性观测终端。它的原则是先展示来源、频率和局限，再展示指标，不输出买卖建议或跨市场黑箱评分。

## 当前覆盖

| 市场 | 核心来源 | 更新方式 | 当前状态 |
| --- | --- | --- | --- |
| 美国 | FINRA Margin Statistics、FRED | GitHub Actions 自动刷新 | 已验证 |
| 日本 | JPX Outstanding Margin Trading by Issue | GitHub Actions 自动刷新 | 已验证 |
| 韩国 | KOFIA FreeSIS、FSS FISIS、韩国央行 ECOS、KSD SEIBro、Naver Finance KRX 指数交叉校验 | GitHub Actions 自动刷新 | 已验证 / 二级行情参照 |

每个韩国 KOFIA 响应与日本 JPX XLS 都会以基准日和 SHA-256 前缀归档到 `data/raw/`。页面的审计抽屉显示完整哈希，并可直接打开已归档的原始快照。

## 指标原则

- 只在同一市场、同一口径内进行历史比较。
- 保证金、信用利差、央行资产和融资买卖余额并非同频数据，页面保留各自发布日期。
- JPX 文件以股数记录融资余额；其买卖比不代表市值，也不能与美国或韩国的金额型指标横向比较。
- 韩国使用 KOFIA 1998 年以来的日序列：投资者存管金、信用交易融资、信用供与、实际反对卖出金额及 KOFIA 公布的强平/未收额比例。R2 与 5 日强平金额各显示 10 年历史分位；不输出通用“爆仓阈值”。
- KOFIA 的历史强平比例为其公布字段；部分日期不等于两个展示金额相除，终端不自行反算或替换其定义。
- 韩国标签页的“专项监控”从同一份 KOFIA 快照展示十年逐日强平路径及信用供与构成（融资、大株、申购贷款、担保融资）。十年数据仅在打开韩国标签时加载；强平长窗口压缩时保留每个时间桶中的最高值，避免把尖峰平均掉。
- “信用供与 / 券商自有资本”将 KOFIA 日频信用供与与 FSS FISIS 季度“证券公司”汇总自有资本相除。它是会计资本容量代理，不是法定净资本比率或监管额度利用率；页面明确分别显示两份数据的基准日。
- KOSPI 与 KOSDAQ 只作为杠杆读数的市场背景：KOFIA FreeSIS 日别市场统计提供十年日频序列，最新值同时与 Naver Finance 指数接口交叉校验。页面保留近一年完整绘图点及较早区间代表点，完整序列归档。它们不是 KRX 原始结算文件，不标为可交易执行价。
- 韩国股市总市值/GDP = 同日 KOFIA KOSPI 与 KOSDAQ 市值之和 / 韩国央行 ECOS 最新完整年度名义 GDP。它将日频市值与年频 GDP 结合，适合观察市场规模位置，不是盈利估值或实时经济规模。
- KSD SEIBro 目前提供杠杆/反向 ETF 的产品覆盖、总发行股数、收盘价、净资产总额和市值；终端以 KSD 公布的净资产总额作为基金规模参照，并展示集合市值相对净资产的偏离。它们不等于净申赎或实际杠杆敞口。
- 自第二个不同基准日的 KSD 快照起，终端会逐只比较总发行股数并显示“发行份额变动”。它不是资金净申赎：拆分、合并、上市或下市等公司行为会影响读数。
- JPX 仅公开当前 XLS；终端从接入当天开始保存每个交易日的文件，不会伪造回填历史。

## 数据更新

`Refresh market snapshots` 工作流在工作日定时运行：

1. 从 FINRA 页面读取最新月度保证金统计，并从 FRED 读取高收益利差和联储资产。
2. 下载 JPX 当日 XLS，按有完整买卖余额的标的聚合，并保留“股数而非市值”的口径。
3. 从 KOFIA FreeSIS 两条公开日序列读取 1998 年以来的市场资金、信用供与与实际反对卖出数据；同时读取 FSS FISIS 的证券公司汇总自有资本。先校验覆盖起点、日期、字段及非负数值，再归档完整响应并生成 R2、资本容量代理及韩国专项监控所需序列。
4. 从 KOFIA FreeSIS 的 KOSPI/KOSDAQ 日别市场服务读取十年日频序列，并以 Naver Finance 指数接口交叉校验最新收盘；同时从韩国央行 ECOS 读取最新完整年度名义 GDP，计算股市总市值/GDP 的十年位置。各响应单独归档，不能将行情参照等同于 KRX 原始结算文件。
5. 从 KSD SEIBro ETF 产品页、单只详情和市场汇总页读取产品类别、总发行股数、收盘价、净资产总额与市值，归档完整响应；连续快照会生成明确标注的发行份额变化。
6. 只在 `src/generated/` 或 `data/raw/` 有变化时提交数据快照。失败时不会覆盖上一个有效快照，GitHub Actions 会失败并留下可审计日志。

本地执行：

```bash
npm install
npm run dev
npm run build
npm run refresh:us
python -m pip install xlrd==2.0.2
npm run refresh:jp
npm run refresh:kr
```
